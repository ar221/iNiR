# Reference Extraction: Huashu Design

## Source

- Link: https://github.com/alchaincyf/huashu-design/tree/master
- Medium: design/code repository and product-style demo material.
- Primary value: process, motion grammar, density discipline, variation workflow, calibration tooling.

## Verdict

Strip-mine it. Do not dress iNiR in it.

Huashu is useful as design-production discipline, not as iNiR's visual identity source.

## What Works

- Density control: tiny labels, sparse metadata, large negative fields, and a restrained warm accent.
- Terminal-to-artifact motif: input or prompt resolves into a composed designed object.
- Motion-system thinking: stage, sprite, easing, and timeline concepts encourage named animation beats.
- Tweaks/calibration panel: palette, type, and density controls map directly to iNiR surface tuning.
- Variation workflow: side-by-side directions with labels reduce single-shot taste drift.

## What Is Costume

- Apple-editorial black voids with elegant product framing.
- iPhone/mockup lifestyle presentation.
- Promo-site posture and AI-tool swagger.
- Dramatic emptiness that works for a deck but not for a daily desktop shell.

## Do Not Copy

- Direct code, assets, music, templates, or styling.
- Product-mockup framing.
- Large editorial canvases where iNiR needs persistent utility.
- Launch-page voice.
- Visual decisions that erase iNiR's Material You glass and shell hierarchy.

## iNiR Compatibility Score

- Retrofuturism: 4/5.
- Modern flair: 4/5.
- Shell utility: 3/5.
- Daily-use survivability: 3/5.
- Nerd-pride factor: 4/5.
- Drift risk: 4/5.

## Extracted Design Moves

### Mission Metadata

- Source observation: Huashu uses small labels and sparse metadata to create technical hierarchy.
- iNiR translation: terminal-hint mono labels for status, time, module identity, and system state.
- Best surfaces: dashboard modules, launcher rows, notifications, workspace overlays.
- Risk: over-labeling can become faux-console clutter.
- Implementation note: reserve uppercase metadata for short, scannable labels.

### Artifact From Command

- Source observation: input or prompt resolves into a polished designed artifact.
- iNiR translation: commands, agent activity, logs, or system state can resolve into composed shell surfaces.
- Best surfaces: launcher, agent cockpit, system summaries, dashboard reports.
- Risk: making shell UI feel like an AI demo.
- Implementation note: keep the output operational, not promotional.

### Timeline Reveal

- Source observation: motion is organized as staged timelines with easing.
- iNiR translation: use named beats: `wake`, `scan`, `lock`, `pulse`, `retire`.
- Best surfaces: launcher, notification stack, dashboard modules, overlays.
- Risk: over-orchestration slows daily use.
- Implementation note: use staged motion only for meaningful state changes.

### Calibration Deck

- Source observation: Huashu exposes visual parameters through a live tweak panel.
- iNiR translation: internal surface for tuning density, blur, accent, radius, grain, and motion tempo.
- Best surfaces: dev/internal shell panel.
- Risk: scope creep into a full public theme editor.
- Implementation note: start ephemeral and internal.

### Three-Vector Variant

- Source observation: Huashu benefits from visible variation and direction comparison.
- iNiR translation: major design changes get `Mission Glass`, `CRT Console`, and `Material Flight Deck` variants.
- Best surfaces: redesign planning, dashboard modules, launcher concepts.
- Risk: too much process for small changes.
- Implementation note: use only when a reference could shift visual language.

## Pilot Recommendation

Build the Calibration Deck first.

It borrows Huashu's most useful process idea without copying the look. It also gives future iNiR design work a practical tuning instrument.

## Implementation Handoff

- Target surface: hidden/internal calibration panel.
- Required tokens: density, blur, opacity, accent temperature, border opacity, radius, grain, motion tempo, mono label weight, clock weight.
- Required states: default, changed, reset, reduced motion.
- Motion beats: `wake` for panel entry, `scan` for preview metadata, `lock` for applied values.
- Accessibility notes: readable labels, keyboard operation, reduced-motion-safe previews.
- Non-goals: public settings, persistence, full theme editor.

## Final Gate

Huashu can make iNiR sharper if treated as a process source. If copied directly, it makes iNiR look like tasteful AI launch collateral.
